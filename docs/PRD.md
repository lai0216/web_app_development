# 專案名稱：食譜收藏系統 (Smart Recipe Vault)

## 1. 專案概述
- **背景與動機**：希望能建立一個專屬的食譜庫，除了能記錄個人喜愛的食譜外，還能藉助 AI 的力量，根據現有食材或偏好自動生成新食譜、提供詳細的作法教學，並能在烹飪過程中解答各種疑難雜症。
- **目標用戶**：我自己（個人使用）。
- **核心價值主張**：打造一個結合「食譜管理」與「AI 烹飪助理」的個人智慧系統，解決每天「不知道吃什麼」和「不會煮」的痛點，提升料理效率與樂趣。

## 2. 功能需求
以下為本系統的主要功能與對應的使用者故事：

1. **生成食譜**
   - 作為 **我自己**，我希望 **能輸入現有的食材或想吃的口味**，以便 **系統能自動發想並生成對應的食譜**。
2. **搜尋食譜**
   - 作為 **我自己**，我希望 **能透過關鍵字（如菜名、食材等）進行搜尋**，以便 **快速找出過去收藏的特定食譜**。
3. **收藏食譜**
   - 作為 **我自己**，我希望 **能把生成的食譜或自己輸入的自創食譜存檔**，以便 **未來想要料理時能隨時查閱**。
4. **生成作法教學說明**
   - 作為 **我自己**，我希望 **能在檢視食譜時點擊生成作法**，以便 **獲得一步步詳細的料理指導與教學說明**。
5. **回答問題**
   - 作為 **我自己**，我希望 **能向系統提問（例如：沒有米酒可以用什麼代替、牛肉煎幾分鐘會熟）**，以便 **及時解決我在料理過程中的突發疑問**。

## 3. 非功能需求
- **技術限制**：
  - 後端架構：Flask (Python)
  - 前端渲染：Jinja2 + 基本的 HTML/CSS/JS
  - 資料庫：SQLite (適合個人專案且易於部署)
  - AI 整合：需串接 LLM API (如 OpenAI API 或 Gemini API) 以前提生成與問答功能。
- **效能與安全考量**：
  - 由於串接 AI 模型可能需要較長時間，前端必須實作 Loading / 載入中 動畫，確保良好的使用者體驗。
  - 確保使用 Parameterized Query 避免 SQL Injection，加強表單的基本檢驗欄位。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have（核心必備重點）**
  - 建立資料庫與基本網站骨架。
  - 食譜的 CRUD（新增、檢視、修改、刪除）。
  - 基本關鍵字搜尋功能。
- **Should Have（主要附加價值）**
  - 串接 AI 提供「生成食譜」與「生成作法教學說明」功能。
- **Nice to Have（進階加分項目）**
  - 獨立的「烹飪問答」聊天視窗。
  - 介面上支援標籤 (Tags) 功能來整理食譜。
