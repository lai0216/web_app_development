# 專案名稱：旅遊規劃系統

## 1. 專案概述
- **背景與動機**：每次規劃旅遊時，常常需要開啟多個視窗來尋找景點、安排路線，同時還需要另外使用試算表來估算和記錄旅遊預算，過程繁瑣且資料分散。因此，希望能開發一個整合行程規劃與預算管理的系統，讓未來的旅遊規劃更加輕鬆有效率。
- **目標用戶**：我自己（個人旅遊規劃者）。
- **核心價值主張**：提供一站式旅遊規劃平台，不僅能順暢安排行程與路線，還能即時追蹤預算，並收藏過去的好行程以供未來參考。

## 2. 功能需求
以下為本系統的主要功能與對應的使用者故事：

1. **搜尋地點**
   - 作為 **我自己**，我希望 **能夠搜尋想去的景點或地點**，以便 **將它們加入我的旅遊清單中**。
2. **規劃路線**
   - 作為 **我自己**，我希望 **能夠將選擇的地點安排成每天的行程路線（最好能支援拖曳排序）**，以便 **清楚知道每天的活動順序**。
3. **計算預算**
   - 作為 **我自己**，我希望 **能夠在行程中為各個項目（如交通、住宿、餐飲、門票）輸入預估費用**，以便 **系統自動幫我計算與追蹤總預算**。
4. **收藏路線**
   - 作為 **我自己**，我希望 **能夠將安排好的行程路線儲存收藏**，以便 **未來可以隨時查看或重複使用這個行程**。
5. **匯出或打包行程**
   - 作為 **我自己**，我希望 **能夠產生一份完整的旅遊清單（包含行程、預算及個人打包項目）**，以便 **在旅途中或出發前做最終確認**。

## 3. 非功能需求
- **技術限制**：
  - 後端架構：Flask (Python)
  - 前端渲染：Jinja2 + HTML/CSS/JS
  - 資料庫：SQLite (適合個人專案且易於開發與備份)
- **效能與安全考量**：
  - 行程路線排序功能若有用到拖曳(Drag & Drop)，前端操作需確保流暢體驗。
  - 需要基本的資料表單驗證（防止空值或不合理的預算輸入），並避免 SQL Injection。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have（核心必備重點）**
  - 建立資料庫與基本網站骨架。
  - 行程與景點的 CRUD（新增、檢視、修改、刪除）。
  - 基本的預算登錄與自動加總功能。
- **Should Have（主要附加價值）**
  - 將景點加入特定天數並排定順序。
  - 收藏/管理不同趟旅程的功能。
- **Nice to Have（進階加分項目）**
  - 視覺化的拖曳排序行程 (Drag & Drop UI)。
  - 整合 Google Maps 顯示實際地圖與路線距離。
  - 匯出行程為 PDF。
